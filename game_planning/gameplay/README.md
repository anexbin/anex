# Gameplay

**Purpose:** Document core mechanics, AI, collisions, and progression logic.

## Why this folder exists
- Centralizes mechanics planning before coding.
- Reduces trial-and-error when building game logic.
- Helps track potential bugs and design flaws early.

## Suggested files
- mechanics.md → Movement, card interactions, attack system
- ai_opponents.md → AI behavior planning
- progression.md → Player leveling, XP, rewards

## Notes
- Include flow diagrams or sketches of game loops.
- Track lessons learned during playtesting.
